Colombian Agricultural Market Intelligence Dashboard 2019–2023

Angie Manuela Gutierrez-Oviedo · Fundación Universitaria del Área Andina · 2023
FRED FAO UN Comtrade SIPSA-DANE Power Query Power BI Executive Report
Abstract
This capstone builds an interactive Power BI dashboard that integrates currency risk, biotechnology trends, and trade geography into a single market intelligence tool for Colombian agricultural markets. Using Power Query for data transformation and Power BI's data model for cross-source relationships, the dashboard connects FRED macroeconomic data, FAO production statistics, UN Comtrade trade flows, and SIPSA-DANE domestic prices into a unified analytical environment with slicers, maps, and drill-through pages.
1.
Research Question

How can an interactive Power BI dashboard integrate currency risk, biotechnology trends, and trade geography into a single market intelligence tool?

The 2020–2022 portfolio produced valuable analyses, but each existed as a separate file — disconnected spreadsheets and SQL exports that required manual navigation. The 2023 challenge is integration: can multiple data sources and analytical perspectives be unified in a single interactive tool that a non-technical stakeholder could use?

3.
Methodology

The capstone applies Power BI's full stack: Power Query for ETL, data modeling for relationships, DAX for calculated measures, and visualization for interactive storytelling.

Step 1 — Power Query ETL pipeline. Each data source is imported, cleaned, and transformed in Power Query: date formats standardized, frequency harmonized, missing values handled, and lookup tables created for consistent coding across sources.

Step 2 — Star-schema data model. A central fact table (price observations) connects to dimension tables (products, markets, dates, countries) through primary-foreign key relationships, enabling cross-source analysis without manual merging.

Step 3 — DAX calculated measures. Custom measures compute volatility (rolling CV), price indices (base-100), growth rates (YoY%), and trade concentration (HHI) as reusable calculations available across all dashboard pages.

Step 4 — Interactive dashboard pages. Five dashboard pages — Price Monitor, Currency Risk, Trade Geography, Biotech Trends, and Executive Summary — each with slicers for time period, product, and country, enabling drill-down exploration.

4.
Key Finding

The dashboard itself is the finding: a functional proof-of-concept that agricultural market intelligence can be delivered through integrated interactive visualization.

Key Finding
The dashboard successfully integrates four data sources into a coherent analytical environment. Key insights surfaced through interactive exploration include: the asymmetric impact of peso depreciation on import-dependent vs. domestically-sourced grains, the geographic concentration of Colombia's agricultural trade in fewer partners than regional peers, and the correlation between biotech adoption rates and yield stability across Latin American countries.

The interactive format enables stakeholder-driven exploration — a non-technical user can filter by product, period, and country to test hypotheses without modifying underlying data or formulas. This represents a fundamental shift from the analyst-as-gatekeeper model of the spreadsheet era.

5.
What I Learned

Building a star-schema data model in Power BI taught me relational data architecture — the same principles that underpin SQL databases. Understanding how fact tables and dimension tables connect through keys fundamentally changed how I think about organizing analytical data.

Writing DAX measures for rolling volatility and price indices introduced a formula language that is conceptually closer to programming than to spreadsheet formulas. The experience of defining reusable calculated measures bridges toward Python function writing in 2024.

Designing interactive dashboards for non-technical audiences taught me to separate the analytical complexity (hidden in the data model and DAX) from the user experience (clean visuals with intuitive filters). This separation of concerns is a design principle that carries through all subsequent work.

6.
Limitation
Tool Limitation
Power BI's statistical capabilities are limited compared to dedicated statistical software. The dashboard can compute descriptive statistics and visualize trends but cannot perform regression analysis, significance testing, or time-series modeling natively. These limitations motivate the transition to Python in 2024.
7.
Deliverable

The deliverable is a Power BI Desktop file (.pbix) containing the complete data model, Power Query transformations, DAX measures, and five interactive dashboard pages. An executive summary PDF is also provided for non-Power BI users.
