Agricultural Commodity Price Forecasting in Emerging Markets: A Python-Based Analysis

Angie Manuela Gutierrez-Oviedo · Fundación Universitaria del Área Andina · 2024
FRED API FAO USDA ISAAA ARIMA Backtesting Jupyter Notebook
Abstract
This capstone builds a Python-based ARIMA forecasting framework for agricultural commodity prices in Colombian and Latin American markets. Using programmatic data retrieval via the FRED API, automated time-series preprocessing with pandas, and formal ARIMA model estimation with statsmodels, the analysis tests whether statistical forecasting models can outperform naive baselines for corn, coffee, and soybean prices. The project represents the portfolio's transition from manual analysis to reproducible, code-based research.
1.
Research Question

Can a Python-based ARIMA forecasting model outperform a naive baseline in predicting agricultural commodity prices in Colombian and Latin American markets?

The 2022 portfolio demonstrated price transmission relationships statistically; the 2023 portfolio visualized them interactively. The natural next question is: can we forecast prices? ARIMA provides a formal forecasting framework that tests whether historical price patterns contain predictive information beyond the most recent observation (the naive benchmark).

Three commodity price series are selected for their relevance to Colombian markets and their different volatility profiles: corn (high import dependence), coffee (primary export), and soybeans (regional benchmark).

3.
Methodology

The capstone follows a standard time-series forecasting workflow: data retrieval, preprocessing, stationarity testing, model estimation, and backtesting.

Step 1 — Automated data retrieval. Python's fredapi library retrieves price series programmatically, with error handling and automatic date alignment. This replaces the manual download-and-copy workflow used in 2020–2023.

Step 2 — Stationarity testing. Augmented Dickey-Fuller (ADF) tests determine whether each price series is stationary or requires differencing. This addresses the non-stationarity limitation identified in the 2022 portfolio.

Step 3 — ARIMA model estimation. Grid search over (p,d,q) parameters identifies the optimal ARIMA specification for each commodity, using AIC/BIC for model selection. Diagnostic plots (residual ACF, QQ-plot) verify model adequacy.

Step 4 — Backtesting framework. An expanding-window backtest computes rolling one-month-ahead forecasts over the 2022–2024 period, comparing ARIMA predictions against a naive (random walk) baseline using RMSE and MAE metrics.

4.
Key Finding

The backtesting results show mixed but informative performance across commodities.

Key Finding
The ARIMA model outperforms the naive baseline for coffee prices (lower RMSE by approximately 10–15%), performs comparably for corn, and slightly underperforms for soybeans. The improvement for coffee is attributed to its more persistent autocorrelation structure (seasonal harvest cycles). The corn and soybean results suggest that these markets are closer to efficient (random walk), consistent with their deep integration into global futures markets where information is rapidly priced in.

The commodity-specific results reinforce a key lesson: no single model works universally. The forecasting value of ARIMA depends on the autocorrelation structure of each series, which in turn reflects the market's efficiency and integration. This motivates the exploration of conditional volatility models (GARCH) in 2025.

5.
What I Learned

The transition to Python was the defining advancement of 2024. Writing code instead of clicking through menus changes the relationship between analyst and data: every step is documented, reproducible, and scalable. The entire analysis can be re-run with a single command — impossible in the spreadsheet and Power BI workflows of prior years.

ADF testing for stationarity formally addresses the limitation identified in the 2022 portfolio: that OLS regression on non-stationary series produces spurious results. The discipline of testing stationarity before modeling is now embedded in all subsequent time-series work.

The backtesting framework — evaluating models on out-of-sample data rather than in-sample fit — introduced the fundamental distinction between fitting and forecasting. A model can fit historical data perfectly and forecast poorly; only backtesting reveals this.

6.
Limitation
Method Limitation
ARIMA models are univariate — they use only the price series' own history for forecasting, ignoring external variables (exchange rates, supply shocks, policy changes) that the 2022 regression analysis showed to be significant predictors. The 2025 portfolio addresses this with multivariate panel regression and GARCH models that incorporate exogenous variables.
