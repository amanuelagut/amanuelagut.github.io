Research Brief 2 2024 · 2015 – 2024 · Python — pandas, matplotlib, FRED API

Automating Commodity Price Time-Series Analysis with Python and FRED API

Angie Manuela Gutierrez-Oviedo · Fundación Universitaria del Área Andina · 2024
FRED API pandas matplotlib Automation Time Series
Abstract
This brief demonstrates how the FRED API and Python can automate the retrieval and time-series analysis of agricultural commodity prices, replacing the manual Excel workflows used in 2020–2023. The brief develops reusable functions for data retrieval, cleaning, transformation, and visualization that serve as building blocks for the 2024 capstone and 2025 working paper. The methodological focus is on reproducibility and automation rather than novel analytical findings.
1.
Research Question

How can the FRED API and Python automate the retrieval and time-series analysis of agricultural commodity prices, replacing manual Excel workflows?

Every analysis in the 2020–2023 portfolio began with manual data download, copy-paste into spreadsheets, and format-specific cleaning. This brief asks: can the entire data pipeline — from source to visualization — be automated in code, making the analysis instantly reproducible and updatable?

2.

3.
Methodology

The brief develops a modular Python pipeline with reusable functions.

Step 1 — API connection and retrieval functions. A wrapper function handles FRED API authentication, series retrieval, error handling, and automatic date filtering. The function accepts a list of series IDs and returns a merged DataFrame.

Step 2 — Data cleaning pipeline. Automated functions for missing value interpolation, frequency conversion (daily to monthly), outlier flagging, and unit standardization. Each step is documented with docstrings and inline comments.

Step 3 — Visualization templates. Reusable matplotlib functions for common chart types: dual-axis time series, scatter with regression line, rolling statistics, and multi-panel comparison. These templates are used throughout the 2024 capstone.

4.
Key Finding

The methodological brief demonstrates that the entire FRED-to-visualization pipeline can be automated in under 200 lines of Python code.

Key Finding
The automated pipeline reduces the time from data retrieval to initial visualization from approximately 2–3 hours (manual workflow) to under 5 minutes (code execution). More importantly, the pipeline is fully reproducible: running the same notebook at any future date automatically retrieves updated data and regenerates all analysis and charts. The modular function design allows any component to be reused or modified independently.

Automation is not just about speed — it is about reproducibility and error reduction. The manual workflows of 2020–2023 were vulnerable to copy-paste errors, version confusion, and undocumented transformations. The code-based approach makes every step explicit and auditable.

5.
What I Learned

API-based data retrieval fundamentally changed my analytical workflow. Instead of downloading files, the data comes programmatically — and this means the analysis is always current, always reproducible, and always documented.

Writing reusable functions with docstrings introduced software engineering principles into research: DRY (Don't Repeat Yourself), separation of concerns, and code documentation. These practices scale directly to the 2025 working paper's codebase.

Building matplotlib visualization templates taught me that good visualization starts with good defaults — pre-configured color palettes, consistent axis formatting, and readable annotations that can be applied to any dataset with minimal customization.

6.
Limitation
Scope Limitation
The brief focuses on FRED data only. Other key sources (SIPSA-DANE, UN Comtrade, FAO) do not have equivalent Python APIs and still require manual download. Building API wrappers or web scrapers for these sources would complete the automation pipeline but is outside the scope of this brief.
